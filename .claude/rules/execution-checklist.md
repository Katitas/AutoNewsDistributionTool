# 実装計画実行チェックリスト

複雑な実装計画を実行する際のチェックリスト。

---

## フェーズ開始時（必須）

- [ ] 計画ドキュメント（docs/plans/）を読み込み済み
- [ ] INTEGRITY_CHECK.md の現在値を確認済み
- [ ] 設計書（docs/design/）の該当セクションを確認済み
- [ ] 環境変数・前提条件の準備完了

---

## フェーズ実行中（各タスク完了時）

- [ ] タスク完了ごとに検証を実行
- [ ] 実装と**同時に**ドキュメント更新（遅延禁止）
- [ ] 発見された課題は**即座に** `docs/issues/` に記録
- [ ] テストが通ることを確認

### 検証手段のバリエーション

タスクの性質に応じて適切な検証を選択する：

| 変更内容 | 検証手段 |
|----------|----------|
| ロジック変更 | ユニットテスト実行 |
| API変更 | 統合テスト or curlで動作確認 |
| UI変更 | ビルド成功確認 + ブラウザでの目視確認 |
| インフラ変更 | `cdk diff` で意図した差分のみか確認 |
| リファクタリング | 全テスト実行 + 期待出力との照合 |

**原則: 「動くはず」ではなく「動いた証拠」を残す**

---

## フェーズ完了時（必須）

- [ ] INTEGRITY_CHECK.md の該当行を更新
- [ ] 設計書に実装内容を反映
- [ ] 全テスト実行
- [ ] PR作成前にコードレビュー実行

---

## 進捗追跡ルール

1. **TaskCreate** でタスクを作成し、**TaskUpdate** で進捗を更新する
2. 各タスク開始時に `status: in_progress` に更新
3. 各タスク完了時に `status: completed` に更新
4. ブロッカー発見時は新規タスクを作成し `blockedBy` で依存関係を設定

---

## 中断・再開時の注意

- 中断前: 現在の進捗状態を TaskList で確認し、メモを残す
- 再開時: TaskList で未完了タスクを確認してから作業再開
- コンテキスト喪失時: 計画書を再読し、完了済みタスクをスキップ

---

## 実行漏れ防止の鉄則

1. **1タスク1コミット** - 細かく区切って進捗を可視化
2. **テスト先行** - 実装前にテストケースを確認
3. **ドキュメント同時更新** - 実装後に「あとで」は禁止
4. **Issue即時記録** - 気づいた課題は忘れる前に記録
