# 実装準備チェックリスト

コード実装タスクを開始する前の必須確認事項。

---

## 1. 既存コード確認（必須）

### 新規ファイル作成時
- [ ] 同名ファイルが存在しないか確認
- [ ] 類似機能の既存実装を確認（パターン参照）
- [ ] import パスを既存コードから確認

### 既存ファイル修正時
- [ ] 修正対象ファイルを**必ず読む**
- [ ] 既存の関数・クラス名を確認
- [ ] 既存のimport文を確認
- [ ] 既存のエラーハンドリングパターンを確認

---

## 2. 関数重複チェック

新規関数を追加する前に、既存ファイルで同名・類似関数がないか確認。

既存関数がある場合の対応:
1. **そのまま使用** - 既存実装で要件を満たす場合
2. **拡張** - 既存関数にパラメータ追加
3. **置換** - 明示的に削除してから新規作成

---

## 3. 設計書のコード例について

設計書のコード例は**参考レベル**として扱う:

1. **そのままコピーしない**
2. 既存コードのパターンを優先
3. import文は既存コードから確認
4. 型ヒントは既存コードのスタイルに合わせる

---

## 4. テスト作成

### テストファイルがない場合
1. 既存テストのパターンを確認
2. テスト設定ファイルを確認
3. モック・フィクスチャのパターンを確認

### テスト実行前
- [ ] テストファイルパスが正しいか確認
- [ ] 必要な依存が存在するか確認

---

## 5. タスク開始時の標準手順

```
1. タスク内容を確認（TaskGet）
2. 関連ファイルを読む（Read）
3. 既存パターンを確認（Grep/Glob）
4. 実装
5. テスト実行
6. 進捗更新（TaskUpdate）
```
