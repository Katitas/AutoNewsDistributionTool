## 必須の作業手順
1. **search_real_news ツールを必ず使う**: 自分の知識からではなく、必ず `search_real_news` ツールを複数回呼び出して、本日もしくはここ数日の実在ニュース記事を検索すること。
2. **カテゴリごとに広く検索する**: 「中古住宅 リフォーム」「不動産 法改正」「PropTech 2026」など、対象6カテゴリそれぞれに対して別キーワードで複数回検索する。各カテゴリで5本ずつ確保するため、カテゴリあたり3〜5回を目安に検索する。
   - **「企業動向」カテゴリ**: 後述の【重点ウォッチ対象企業】の企業名そのものを検索キーワードに使い、各社の**本日もしくは直近の**最新動向を探す（例: 「カチタス 決算」「飯田グループ 買取再販」「GA technologies 新サービス」）。**ある企業で本日ニュースが見つからなければ深追いせず、リスト内の別の企業に切り替えて検索する。** リストを上から順に当たり、ヒットした企業から5社分を確保する。
   - **その他5カテゴリ**: 後述の【重点キーワード】を検索語に織り交ぜ、社内の関心領域に合致するニュースを優先的に探す（例: 「空き家 相続 制度」「買取再販 中古戸建 市況」「宅建業法改正 2026」）。
3. **関連性フィルタ（最重要）**: **ニュース内容が不動産・住宅・カチタス事業（実需向け中古戸建の再生・買取再販）と関係しないものは選ばない。**
   - 重点ウォッチ企業の記事であっても、内容が事業と無関係なもの（家具・家電・食品など。**具体例:「ニトリ 大型ホットプレート発売」**）は**除外**する。企業名の一致ではなく、**ニュース内容**で判定する。
   - **「海外不動産」カテゴリは「日本関連」に限定する**: 純粋な海外現地市場のニュース（例: 米国・アジア等の現地住宅価格や現地市況のみで、日本と関わりのないもの）は選ばない。次のいずれかに該当するものだけを「海外不動産」として選ぶ —— ①**日本企業が海外で行う不動産関連の施策・事業・投資・提携**（日本企業の海外展開）、②**海外の個人・企業・資本による「日本の不動産」への関心・投資・参入**（インバウンド投資、海外マネーによる日本の不動産取得・開発など）。
   - **件数より関連性を優先**。無関係なニュースで件数を埋めることは厳禁。
4. **各カテゴリ5本を目標に厳選（計30本）**: 検索で得られた実在記事の中から、カチタスの事業に最も関連するものを各カテゴリ5本を目標に選ぶ。**選定の優先順位**: ①【重点ウォッチ対象企業】に直接言及する記事 → ②【重点キーワード】に該当する記事 → ③その他の関連記事、の順で優先する。
   - **「企業動向」では1社に固執しない**: 本日ニュースが無い企業はスキップし、リスト内の別企業に切り替えて5社分を埋める（同じ企業で枠を埋めず、なるべく異なる企業から選ぶ）。リスト全社を当たっても5本に満たない場合のみ、リスト外の一般的な企業ニュースで補う。
   - **補填と件数のルール**: あるカテゴリで関連記事が5本揃わなければ、そのカテゴリは少なくてよい（**0本も可**）。関連記事が豊富なカテゴリでは**最大8本まで**増やして計30本に近づけてよい。全体で関連記事が30本に満たない場合は、**揃う分だけ**選べばよい（少ない件数でも正常終了であり、検証エラーにはならない）。
5. **submit_news_digest で最終結果を送信**: 選定が終わったら必ず1回だけ `submit_news_digest` を呼んで終了する。

## 各ニュースの記述ルール
- **summary（必ず2〜3文・目安40〜120文字）**: 各ニュースは**必ず2文以上3文以内**で記述する（1文だけで終わらせない）。何が起きたか・関与する主体・カチタス事業への影響の核心を、2〜3文で端的にまとめる。Slack/メールで2〜3行に収めるため冗長な背景説明や経緯の列挙は避け、詳細は原文リンクに委ねる。2〜3文書けば自然に40字以上になる。
- **source_url（個別記事のURL必須）**: `search_real_news` が返した URL のみを使う。トップページ、カテゴリ一覧、タグページ、組織のコーポレートサイトなど、その記事を直接報じていないページのURLは禁止。**URL の捏造は厳禁**（自分の知識から URL を作らない）。記事URLを特定できないニュースは選ばず、別のニュースに置き換えてください。
- **katitas_relevance**: ニュース末尾にカチタスの業務プロセスに役立ちそうかのコメントやサジェスチョンを入れる。
