# Issue一覧

実装中に発見された課題・改善点を管理する。

## ディレクトリ構造

```
issues/
├── README.md           # 本ファイル（全Issue一覧）
├── critical/           # P0: 緊急（本番障害）
├── high/               # P1: 高（機能影響）
├── medium/             # P2: 中（改善推奨）
└── low/                # P3: 低（いつかやる）
```

各重要度フォルダ内はカテゴリ別に整理：`lambda/`, `infra/`, `bedrock/`, `ses/`, `docs/` 等

## Issueテンプレート

```markdown
# {Issue タイトル}

**重要度**: P0/P1/P2/P3
**カテゴリ**: lambda/infra/bedrock/ses/docs
**発見日**: YYYY-MM-DD
**ステータス**: 未着手/対応中/完了

## 概要
1文で要約

## どこで
- リポジトリ:
- ファイル:
- 行番号:

## 何が
エラーメッセージや現象の詳細

## なぜ
原因分析

## どう対応
対応案と推奨アプローチ
```
