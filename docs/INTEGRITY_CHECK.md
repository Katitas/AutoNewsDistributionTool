# 設計書-実装 整合性チェック

設計書と実装コードの整合性を管理する。
実装変更後は該当行のステータスを更新すること。

**最終チェック日**: 2026-05-01

---

## チェック方法

| ステータス | 意味 |
|-----------|------|
| ✅ | 設計書と実装が一致 |
| ⚠️ | 一部乖離あり（Issue記録済み） |
| ❌ | 未実装 or 大幅乖離 |
| - | 該当なし |

---

## 運用ルール

### いつチェックするか

| タイミング | アクション |
|-----------|-----------|
| 実装フェーズ開始時 | この表の現在値を確認し、乖離がないか把握 |
| 実装フェーズ完了時 | 変更した機能に対応する行のステータスを更新 |
| PR作成前 | ⚠️/❌ の行がないか最終確認 |
| 定期（月次推奨） | 全行を棚卸し、最終確認日を更新 |

### ステータス更新の判断基準

| 状況 | ステータス |
|------|-----------|
| 設計書の通りに実装されている | ✅ |
| 実装が設計書と異なるが、Issueに記録済み | ⚠️ |
| 設計書が未作成、または大幅に異なる | ❌ |
| 設計書を修正して実装と一致させた | ✅（設計書の変更履歴にも記録） |

## 乖離発見時の対応フロー

```
乖離を発見
    ↓
設計書とコード、どちらが正しいか判断
    ↓
┌─ 設計書が正しい → コードを修正
│
├─ コードが正しい → 設計書を更新 + 変更履歴を追記
│
└─ 判断できない → docs/issues/ にIssue作成（P1以上）
    ↓
この表のステータスを更新
```
